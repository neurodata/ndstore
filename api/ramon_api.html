RAMON API¶

JSON Service¶

The RAMON JSON service retrieves RAMON objects from the database and returns them in JSON format.

getJSONObject¶

GET (string: server_name)/nd/ramon/(string: token_name)/(string: channel_name)/(int: id)/¶
Synopsis:

Retrieve the RAMON object specified by ID as a JSON object.
Parameters:
  • server_name (string) – NeuroData Server Name (typically cloud.neurodata.io
  • token_name (string) – NeuroData Token
  • channel_name (string) – NeuroData Channel
  • id (int) – An integer identifier for the request RAMON object.
Status Codes:
  • 200 OK – RAMON object returned as JSON.
  • 404 Not Found – RAMON object not found or error.

Example Return:

{
   "21": {
         "syn_weight": 0.0,
         "ann_type": 2,
         "syn_type": 0,
         "ann_status": 0,
         "postgaba": "0",
         "ann_confidence": 1.0,
         "ann_id": 21,
         "syn_centroid": "[]",
         "ann_author": "Randal Burns",
         "gaba": "0",
         "display": "0",
         "syn_segments": [0]
      }
}

getBoundingBox¶

GET (string: server_name)/nd/ramon/(string: token_name)/(string: channel_name)/(int: id)/boundingbox/(int: res)/¶
Synopsis:

Retrieve the bounding box around the specified RAMON object along with the rest of the RAMON metadata and return it as a JSON object.
Parameters:
  • server_name (string) – NeuroData Server Name (typically cloud.neurodata.io
  • token_name (string) – NeuroData Token
  • channel_name (string) – NeuroData Channel
  • id (int) – An integer identifier for the request RAMON object.
  • res (int) – Resolution for Bounding Box Coordinates
Status Codes:
  • 200 OK – RAMON object and bounding box returned as JSON.
  • 404 Not Found – RAMON object not found or error.

Example Return:

{
   "21": {
      "syn_weight": 0.0,
      "ann_type": 2,
      "syn_type": 0,
      "ann_status": 0,
      "postgaba": "0",
      "ann_confidence": 1.0,
      "ann_id": 21,
      "syn_centroid": "[]",
      "bbcorner": [1549, 2940, 0],
      "ann_author": "Forrest Collman",
      "bbdim": [112, 101, 4],
      "gaba": "0",
      "display": "0",
      "syn_segments": [0]
   }
}

Bounding Box:

The bounding box is described by the bbcorner and bbdim keys:

  • bbcorner: The lower corner of the bounding box.
  • bbdim: The dimensions of the bounding box.

To reconstruct the upper corner of the bounding box, add bbcorner and bbdim.

Query by Key¶

query¶

GET (string: server_name)/nd/ramon/(string: token_name)/(string: channel_name)/query/(string/int/float: key)/(string/int/float: value)/¶
Synopsis:

Retrieve a list of RAMON objects with the specified key/value combination.
Parameters:
  • server_name (string) – NeuroData Server Name (typically cloud.neurodata.io
  • token_name (string) – NeuroData Token
  • channel_name (string) – NeuroData Channel
  • key (string/int/float) – The RAMON object key to search on.
  • value (string/int/float) – The RAMON object value to match for the specified key.
Status Codes:
  • 200 OK – RAMON object list returned.
  • 500 Internal Server Error – Server Error (text returned).

Note:

In the RAMON data model all standard RAMON attributes are also keys. So, finding all synapses would amount to using the query interface with a key of ann_type and a value of 2.

topKeys¶

GET (string: server_name)/nd/ramon/(string: token_name)/(string: channel_name)/topkeys/(int: num_results)/¶
Synopsis:

Retrieve a list of the top RAMON keys for the specified project.
Parameters:
  • server_name (string) – NeuroData Server Name (typically cloud.neurodata.io
  • token_name (string) – NeuroData Token
  • channel_name (string) – NeuroData Channel
  • num_results (int) – The number of keys to return.
Status Codes:
  • 200 OK – List of RAMON keys returned.
  • 500 Internal Server Error – Server Error (error text returned).

topKeysByType¶

GET (string: server_name)/nd/ramon/(string: token_name)/(string: channel_name)/topkeys/(int: num_results)/type/(int: ramon_type)/¶
Synopsis:

Retrieve a list of the top RAMON keys for the specified project restricted to a certain type of RAMON object.
Parameters:
  • server_name (string) – NeuroData Server Name (typically cloud.neurodata.io
  • token_name (string) – NeuroData Token
  • channel_name (string) – NeuroData Channel
  • num_results (int) – The number of keys to return.
  • ramon_type (int) – Restrict the top keys list to this type of RAMON object.
Status Codes:
  • 200 OK – List of RAMON keys returned.
  • 500 Internal Server Error – Server Error (error text returned).

Field Service¶

setField¶

GET (string: server_name)/nd/ca/(string: token_name)/(string: channel_name)/setField/(string: ramon_field)/(string/int/float: ramon_value)¶
Synopsis:

Set the value of the RAMON field for the specified channel
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is cloud.neurodata.io
  • token_name (string) – Token Name in NeuroData.
  • channel_name (string) – Channel Name in NeuroData. Optional. If missing will use default channel for the token.
  • ramon_field (string) – RAMON field. For more details look at the RAMON documents.
  • ramon_value (string/int/float) – Value of the corresponding RAMON field
Status Codes:
  • 200 OK – No error
  • 404 Not Found – Error in the syntax or file format

Annotation Service¶

GET¶

GET (string: server_name)/nd/ca/(string: token_name)/(string: channel_name)/(int: annotation_id)/(string: option_args)/(int: resolution)/¶
Synopsis:

Get an annotation from the server
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is cloud.neurodata.io
  • token_name (string) – Token Name in NeuroData.
  • channel_name (string) – Channel Name in NeuroData. Optional. If missing will use default channel for the token.
  • annotation_id (int) – Id of the annotation to be cut from the database.
  • options_args (string) – Optional Arguments. This can be overwrite, preserve, exception.
  • resolution (int) – Resolution for the annotation
Status Codes:
  • 200 OK – No error
  • 404 Not Found – Error in the syntax or file format

Resources API¶

Dataset¶

POST (string: server_name)/nd/resource/dataset/¶
Synopsis:

Create a dataset
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is cloud.neurodata.io
Status Codes:
  • 201 Created – Resource created
  • 400 Bad Request – Bad Request. Incorrect syntax
  • 403 Forbidden – Forbidden. Insufficient permissions.

Example Request:

POST /nd/resource/kasthuri11/ HTTPS/1.1
Host: cloud.neurodata.io
Content-Type: application/json

{
  "dataset_name" : "kasthuri11",
  "ximagesize" : 2000,
  "yimagesize" : 2000,
  "zimagesize" : 100,
  "xvoxelres" : 5.0,
  "yvoxelres" : 5.0,
  "zvoxelres" : 10.0,
  "public" : 1
}

Example Responses:

HTTPS/1.1 201 Created
Content-Type: text/plain

HTTPS/1.1 400 BadRequest
Content-Type: text/plain

HTTPS/1.1 403 Forbidden
Content-Type: text/plain
GET (string: server_name)/nd/resource/dataset/¶
Synopsis:

List all datasets
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is cloud.neurodata.io
Status Codes:
  • 200 OK – OK. List of datasets returned
  • 400 Bad Request – Bad Request. Incorrect syntax
  • 403 Forbidden – Forbidden. Insufficient permissions.

Example Request:

GET /nd/resource/kasthuri11/ HTTPS/1.1
Host: cloud.neurodata.io
Content-Type: text/plain

Example Responses:

HTTPS/1.1 200 OK
Content-Type: application/json

{
  'kasthuri11'
  'bock11'
  'lee15'
}

HTTPS/1.1 400 BadRequest
Content-Type: text/plain

HTTPS/1.1 403 Forbidden
Content-Type: text/plain

Channel¶

POST (string: server_name)/nd/sd/dataset/(string: dataset_name)/project/(string: project_name)/token/(string: token_name)/¶
Synopsis:

Create a list of channels for an existing project using the project token and JSON file.
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is openconnecto.me.
  • token_name (string) – Token Name in NeuroData.
Status Codes:
  • 200 OK – No error
  • 400 Bad Request – Error in file format or if channel already exists
  • 404 Not Found – Error in the syntax

Example Request:

POST /nd/ca/test_kat1/ HTTPS/1.1
Host: openconnecto.me
Content-Type: application/json

{
  "channels": {
    "CHAN1": {
      "channel_name": "CHAN1",
      "channel_type": "image",
      "datatype": "uint8",
      "readonly": 0
    },
    "CHAN2": {
      "channel_name": "CHAN2",
      "channel_type": "image",
      "datatype": "uint8",
      "readonly": 0
    }
  }
}

Example Responses:

HTTPS/1.1 201 Created
Content-Type: text/plain

HTTPS/1.1 400 BadRequest
Content-Type: text/plain

HTTPS/1.1 403 Forbidden
Content-Type: text/plain

deleteChannel¶

POST (string: server_name)/nd/ca/(string: token_name)/deleteChannel/¶
Synopsis:

Delete a list of channels for an existing project using the project token and a JSON file.
Parameters:
  • server_name (string) – Server Name in NeuroData. In the general case this is openconnecto.me.
  • token_name (string) – Token Name in NeuroData.
Status Codes:
  • 200 OK – No error
  • 400 Bad Request – Error in file format or if channel does not exist
  • 404 Not Found – Error in the syntax

Example Request:

POST /nd/ca/test_kat1/ HTTPS/1.1
Host: openconnecto.me
Content-Type: application/json

{
  "channels": [
             "CHAN2",
             "CHAN3"
  ]
}

Example Responses:

HTTPS/1.1 200 OK
Content-Type: text/plain

Success. Channels deleted.

HTTPS/1.1 400 BadRequest
Content-Type: text/plain

Missing. Required fields.

HTTPS/1.1 400 BadRequest
Content-Type: text/plain

Error saving models. The channels were not deleted.
